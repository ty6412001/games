# week 题库选择 + week-02 扩容设计

**目标**

在现有开始游戏流程中补充 `week` 题库选择能力，并将 `week-02` 从 `55` 题扩展到 `95` 题，使其和 `week-01` 一样成为更完整的教材进度包。

**用户要求**

- 在界面里可选择使用哪一周的题库
- 当前至少支持明确选择 `week-01` 和 `week-02`
- `week-02` 扩展到 `95` 题
- 数学范围对齐苏教版目录 `1-5`，以最新单元为主，并补充前置单元复习
- 语文范围对齐苏教版一下 `1-4` 单元全部内容，以最新单元为主，并补充前置单元复习
- 英语继续保留 `15` 题，按现有教材口径，并带一部分前置复习

**范围**

- 修改开始游戏 / setup 流程，加入 `week` 选择
- 重写 `apps/web/public/question-packs/week-02.json`
- 补充与 `week` 选择和 `week-02` 扩容相关的测试

**非目标**

- 不重做完整的题库管理页面
- 不调整 `week-03` 到 `week-10` 的题量
- 不修改独立 `brain-pack`
- 不改变现有游戏答题主流程

**方案选择**

采用“同时补入口 + 重做 week-02”的方案。

理由：

- 只扩 `week-02` 不补入口，用户无法明确选周，体验不完整
- 一次把入口和 `week-02` 数据口径对齐，后续继续扩其它周时可以复用同一套选择逻辑

**week 选择入口**

- 不新增新页面
- 直接放在现有开始游戏 / setup 页面中
- 默认选中 `week-01`
- 本次交互至少明确支持切换 `week-01` 与 `week-02`
- 实现上应基于现有题库索引渲染，避免把逻辑写死在 UI 中

说明：

- 即使界面这次优先突出 `week-01` 和 `week-02`，底层结构仍应允许后续扩展更多周

**week-02 题量结构**

- 总题量固定为 `95`
- 数学 `40`
- 语文 `40`
- 英语 `15`

**week-02 数学内容**

- 对齐苏教版目录 `1-5`
- 以最新单元内容为主，前置单元作复习补充

建议权重：

- 目录 `4-5`：约 `24-26` 题
- 目录 `1-3`：约 `14-16` 题

要求：

- 仍以 `choice` 为主
- `input` 题只允许唯一答案
- 不做机械重复换数字的低质量堆题

**week-02 语文内容**

- 对齐苏教版一下 `1-4` 单元全部内容
- 以最新单元为主，前面单元为复习补充

建议权重：

- 第 `4` 单元：约 `16-18` 题
- 第 `3` 单元：约 `10-12` 题
- 第 `1-2` 单元：约 `10-12` 题

要求：

- 课文、识字、词句理解、单元练习能力混合覆盖
- 题干不重复
- 不混入 `brain` 题

**week-02 英语内容**

- 保持 `15` 题
- 延续当前英语教材口径
- 以最新单元为主，前置单元做复习补充

**题型规则**

- 继续只使用现有支持的题型
- 但本次仍以 `choice` 为主
- `input` 只保留唯一答案
- 不引入开放式答案
- 不引入重复项排序题

**验证**

- 新增或更新测试，验证 setup 中可选择 `week`
- 新增 `week-02` 数据测试，验证：
  - 总数 `95`
  - `math 40 / chinese 40 / english 15`
  - 不含 `brain`
- 运行：
  - `pnpm --filter @ultraman/web run test`
  - `pnpm --filter @ultraman/web run build`
