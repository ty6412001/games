# iPad 横屏布局适配设计

**目标**

修正部署后的游戏页面在 `iPad` 横屏浏览器中的可视区域溢出问题，确保首屏主要交互完整可见，不再出现底部内容被裁切但页面又无法滚动的情况。

**问题定位**

当前游戏主界面在 [GameScreen.tsx](/Users/zouhui/Documents/code/games/apps/web/src/features/game/GameScreen.tsx:1) 中采用了固定整屏高度和固定三栏布局：

- 外层使用 `h-[100svh]`
- 外层同时使用 `overflow-hidden`
- 中间棋盘区使用固定三栏中的剩余宽度
- 棋盘容器使用 `aspect-square h-full max-h-full max-w-full`

这套组合在 `iPad` 横屏这类“宽度足够但高度有限”的视口下会导致棋盘和侧栏共同争抢高度，最终把底部内容挤出视口，而且因为外层禁止滚动，用户无法看到被裁切的内容。

**范围**

- 修改 `apps/web/src/features/game/GameScreen.tsx`
- 修改 `apps/web/src/features/game/CurrentPlayerSpotlight.tsx`
- 修改 `apps/web/src/features/game/Leaderboard.tsx`
- 补充或调整与视口适配相关的测试

**非目标**

- 不重做桌面端整体视觉结构
- 不改游戏流程、棋盘规则、弹窗逻辑
- 不改题库、题目、战斗或结算逻辑
- 不单独为手机竖屏设计新界面

**方案选择**

采用“横屏平板专门适配”方案，而不是仅靠取消 `overflow-hidden` 兜底。

理由：

- 单纯放开整页滚动只能避免“看不到”，不能解决 `iPad` 横屏下首屏交互拥挤的问题
- 当前三栏结构本身是合理的，适合保留；问题在于尺寸分配策略过硬
- 针对 `md/lg` 之间的平板横屏档位调整尺寸，能在不影响桌面端的前提下修正该问题

**布局策略**

- `桌面端` 继续保留三栏布局：`当前玩家区 / 棋盘区 / 排行榜`
- `iPad` 横屏仍保留三栏，但不再使用当前“左右栏固定宽度、中间区吃剩余空间”的硬分配
- 页面外层从固定整屏裁切改为：
  - 允许页面垂直兜底滚动
  - 侧栏内部可局部滚动
  - 棋盘区域不再强制撑满父容器高度

**棋盘区策略**

- 棋盘必须继续保持正方形
- 在 `iPad` 横屏视口下，棋盘尺寸以“可用高度上限”为优先约束，而不是默认铺满中间列高度
- 棋盘同时仍受中间列可用宽度约束，最终尺寸等于“高度上限”和“中间列宽度上限”中的较小值
- 这样可以让棋盘主动缩小一点，换取整页内容完整可见

**左侧当前玩家区策略**

- 保持现有信息结构和骰子区位置
- 去掉必须撑满整列高度的强约束，让卡片可以在平板横屏下按可用高度收缩
- 底部骰子区保持优先可见，中间信息区在空间不足时让步

**右侧排行榜策略**

- 保持顶部标题和底部“上次骰”固定可见
- 中间列表继续使用 `overflow-y-auto`
- 当高度不足时，只滚动排行榜列表，不让整页被裁切

**断点策略**

- 调整重点放在 `md` 到 `lg` 之间的横屏平板区间
- 大桌面继续保留当前三栏信息密度
- 不为了平板横屏去降低桌面端棋盘尺寸或弱化侧栏内容

**测试与验收**

功能验收标准：

- `iPad` 横屏等价视口下，棋盘、当前玩家信息卡、掷骰按钮、排行榜底部都能进入视口
- 页面不再出现“底部被裁切且无法滚动”的状态
- 侧栏在内容超出时允许局部滚动
- 桌面端原有布局不明显回退

验证方式：

- 更新或新增 Playwright 视口测试，覆盖 `iPad` 横屏等价视口
- 运行 `pnpm --filter @ultraman/web run test`
- 运行 `pnpm --filter @ultraman/web run build`
