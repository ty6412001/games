# 通用题库替换设计

**目标**

将当前独立的 `brain-pack` 从“脑筋急转弯题库”调整为“通用题库”，保留现有独立 pack 加载机制，但重写题目内容与 UI 文案，使其成为一份面向亲子共玩的综合扩展题库。

**用户要求**

- 现有“脑筋急转弯”入口改成“通用题库”
- 题库中去掉偏“数字脑筋题”的内容
- 补充：
  - `十万个为什么`
  - `历史知识`
  - 其它扩展类题目
- 保留简单和困难混合的题目
- 继续放在现有独立 pack 中，通过当前随机出题系统抽取

**范围**

- 重写 `apps/web/public/question-packs/brain-pack.json`
- 更新与该题库相关的数据测试
- 更新 UI 上与 `brain` 对应的展示文案

**非目标**

- 不新增新的 `subject`
- 不修改 `SubjectSchema` 中 `brain` 的底层值
- 不修改 `week-01` 到 `week-10` 周题库
- 不重做现有随机出题主流程

**方案选择**

采用“保留 `subject=brain`，但内容和文案替换成通用题库”的方案。

理由：

- 如果直接把底层 `brain` 改成新的 subject，会牵连 schema、store、加载逻辑、测试与旧数据
- 当前用户需求本质上是“替换内容和展示语义”，不是重做题型系统
- 保留底层字段不变，能把改动集中在题库内容和 UI 文案上

**数据结构**

- 保留现有独立文件：
  - `apps/web/public/question-packs/brain-pack.json`
- 保留其作为独立 pack 被 `loadBrainPack()` 加载的方式
- 文件总题量继续固定为 `100`
- `subject` 继续使用 `brain`

**题库定位**

这份题库不再限定为“脑筋急转弯”，而是一份综合扩展题库。

对用户展示时统一命名为：

- `通用题库`

**内容分类**

题目内容至少覆盖以下五类：

- `脑筋急转弯`
- `十万个为什么`
- `历史知识`
- `生活常识`
- `百科扩展`

要求：

- 去掉偏“数字脑筋题”的内容
- 保留一部分真正的语言/逻辑型脑筋急转弯
- 内容整体更像综合知识问答，而不是纯谜语或纯算数陷阱

**题型与难度**

- 总数 `100`
- 仍以 `choice` 为主
- `input` 只保留少量唯一答案题
- 不引入开放式主观题

建议难度分布：

- `difficulty 1`：约 `45`
- `difficulty 2`：约 `35`
- `difficulty 3`：约 `20`

**质量约束**

- 题干唯一
- `choice` 题保持 `4` 个选项
- `choice.answer` 必须存在于 `options`
- `choice.options` 不允许重复
- `input` 题答案必须唯一
- 不使用过冷、过偏、过成人化的内容

**UI 文案**

需要把用户可见文案从：

- `脑筋急转弯`

替换为：

- `通用题库`

包括但不限于：

- 学科选择卡片文案
- 题目徽章文案

说明：

- 仅改展示文案
- 不改底层 `subject: "brain"` 的内部实现

**验证**

- `brain-pack.json` 继续通过现有题目 schema 校验
- 数据测试应验证：
  - 总数 `100`
  - 全部为 `brain`
  - 不再含“数字脑筋题”类内容
  - 覆盖新的 topic 分类
  - 题干唯一
  - `choice` 题选项和答案合法
- 跑相关 UI 测试
- 运行：
  - `pnpm --filter @ultraman/web run test`
  - `pnpm --filter @ultraman/web run build`
